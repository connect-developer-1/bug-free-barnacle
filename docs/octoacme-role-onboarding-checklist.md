# OctoAcme Role-Based Onboarding Checklist

## Purpose
This document provides role-specific onboarding checklists to help new team members quickly ramp up and understand their responsibilities within OctoAcme projects.

---

## Developer Onboarding

### Week 1: Environment & Access
- [ ] Set up development environment and tooling
- [ ] Obtain repository access and clone codebase
- [ ] Review project README and technical architecture docs
- [ ] Set up CI/CD pipelines and understand deployment process
- [ ] Join team Slack/communication channels

### Week 2: Code & Process
- [ ] Complete first "good first issue" or bug fix
- [ ] Participate in code review (as reviewer and author)
- [ ] Attend daily standup and sprint planning
- [ ] Review Definition of Done and coding standards
- [ ] Meet with Project Manager to understand project timeline

### Week 3: Collaboration
- [ ] Shadow or pair with senior developer
- [ ] Attend design review or technical planning session
- [ ] Review acceptance criteria with Product Manager
- [ ] Coordinate with QA on test coverage for a feature
- [ ] Contribute to retrospective discussion

---

## Product Manager Onboarding

### Week 1: Context & Vision
- [ ] Review product vision, roadmap, and OKRs
- [ ] Meet with stakeholders to understand business goals
- [ ] Review user research findings and customer feedback
- [ ] Study success metrics and current analytics dashboards
- [ ] Review Project Charter and One-pager template

### Week 2: Backlog & Planning
- [ ] Review and prioritize backlog with team
- [ ] Attend sprint planning and understand estimation process
- [ ] Review acceptance criteria standards
- [ ] Meet with Data Analyst to define measurement framework
- [ ] Collaborate with UX Designer on user flows

### Week 3: Cross-Functional Collaboration
- [ ] Present roadmap update to stakeholders
- [ ] Participate in demo/review session
- [ ] Coordinate with Project Manager on timeline adjustments
- [ ] Review compliance requirements with Legal/Compliance Advisor
- [ ] Join retrospective and contribute to continuous improvement

---

## Project Manager Onboarding

### Week 1: Project Context
- [ ] Review project charter, timeline, and key milestones
- [ ] Meet with stakeholders and understand communication needs
- [ ] Review risk register and dependency tracking
- [ ] Understand escalation paths and decision-making processes
- [ ] Access project board and status reporting tools

### Week 2: Team & Process
- [ ] Shadow or co-facilitate daily standup
- [ ] Review sprint/iteration process with Agile Coach (if applicable)
- [ ] Meet individually with core team members
- [ ] Understand team capacity and resource constraints
- [ ] Review past retrospectives and lessons learned

### Week 3: Delivery Coordination
- [ ] Lead weekly sync between PM and PdM
- [ ] Facilitate or co-facilitate sprint planning
- [ ] Update stakeholder status report
- [ ] Coordinate cross-team dependencies
- [ ] Review quality gates with QA Engineers

---

## QA Engineer Onboarding

### Week 1: Testing Environment
- [ ] Set up test environments and access credentials
- [ ] Review test strategy and automation framework
- [ ] Understand CI/CD test integration
- [ ] Access bug tracking and test management tools
- [ ] Review acceptance criteria standards

### Week 2: Test Planning
- [ ] Review test plans for upcoming features
- [ ] Participate in sprint planning to understand scope
- [ ] Write first test cases or automated tests
- [ ] Shadow experienced QA on exploratory testing
- [ ] Meet with Developers to discuss testability

### Week 3: Quality Collaboration
- [ ] Execute test plan and report defects
- [ ] Participate in feature demo/review
- [ ] Collaborate with UX Designer on usability testing
- [ ] Review quality metrics with Project Manager
- [ ] Contribute to retrospective on quality improvements

---

## UX Designer Onboarding

### Week 1: Product & User Context
- [ ] Review product vision, user personas, and research findings
- [ ] Access design tools (Figma, Sketch, etc.) and component libraries
- [ ] Review existing design system and style guides
- [ ] Meet with Product Manager to understand roadmap priorities
- [ ] Study analytics data with Data Analyst

### Week 2: Design Process
- [ ] Review design workflow and handoff process
- [ ] Participate in user research or usability testing session
- [ ] Create first design mockup or wireframe
- [ ] Present design concepts in design review
- [ ] Collaborate with Developers on implementation feasibility

### Week 3: Cross-Functional Collaboration
- [ ] Deliver design specs and assets to Developers
- [ ] Participate in sprint planning with acceptance criteria input
- [ ] Coordinate with QA on usability test plans
- [ ] Validate implemented features against design specs
- [ ] Join retrospective to improve design handoff process

---

## Data Analyst Onboarding

### Week 1: Data Infrastructure
- [ ] Set up access to analytics tools and data warehouse
- [ ] Review data schema, instrumentation, and tracking plan
- [ ] Understand key metrics and existing dashboards
- [ ] Meet with Product Manager to understand success criteria
- [ ] Review data quality standards and governance policies

### Week 2: Analysis & Reporting
- [ ] Create or update dashboard for project metrics
- [ ] Conduct first analysis on feature performance or user behavior
- [ ] Attend sprint planning to understand upcoming instrumentation needs
- [ ] Collaborate with Developers on tracking implementation
- [ ] Review A/B testing process and experiment framework

### Week 3: Insights & Impact
- [ ] Present data insights to Product Manager and stakeholders
- [ ] Work with UX Designer to support usability testing with data
- [ ] Identify trends or anomalies and flag to team
- [ ] Participate in retrospective with data-driven recommendations
- [ ] Document analytics processes for future reference

---

## DevOps Engineer Onboarding

### Week 1: Infrastructure & Tools
- [ ] Set up access to cloud infrastructure and CI/CD systems
- [ ] Review infrastructure as code (IaC) and deployment pipelines
- [ ] Understand monitoring, logging, and alerting setup
- [ ] Review security policies and compliance requirements
- [ ] Meet with Legal/Compliance Advisor on audit logging needs

### Week 2: Deployment & Operations
- [ ] Shadow production deployment or release process
- [ ] Review incident response procedures and on-call rotation
- [ ] Understand capacity planning and resource management
- [ ] Collaborate with Developers on build/deployment issues
- [ ] Set up test environments for QA Engineers

### Week 3: Automation & Improvement
- [ ] Implement first infrastructure improvement or automation
- [ ] Review SLO/SLA reports with stakeholders
- [ ] Participate in postmortem or incident review
- [ ] Optimize CI/CD pipeline for faster feedback
- [ ] Join retrospective to discuss infrastructure improvements

---

## Agile Coach Onboarding

### Week 1: Team Assessment
- [ ] Observe team ceremonies (standups, planning, retrospectives)
- [ ] Meet individually with team members to understand dynamics
- [ ] Review team velocity, burndown, and health metrics
- [ ] Understand organizational agile maturity and goals
- [ ] Review past retrospective action items

### Week 2: Coaching & Facilitation
- [ ] Co-facilitate or observe retrospective
- [ ] Provide feedback on ceremony effectiveness
- [ ] Identify impediments and work with Project Manager to remove them
- [ ] Coach team on agile principles and best practices
- [ ] Meet with Product Manager to improve backlog refinement

### Week 3: Continuous Improvement
- [ ] Facilitate team health discussion or improvement workshop
- [ ] Introduce new agile practice or technique (if applicable)
- [ ] Create team maturity assessment and improvement roadmap
- [ ] Collaborate with stakeholders on managing agile expectations
- [ ] Document coaching plan and success metrics

---

## Legal/Compliance Advisor Onboarding

### Week 1: Compliance Context
- [ ] Review applicable regulations (GDPR, CCPA, SOC 2, etc.)
- [ ] Understand company compliance policies and standards
- [ ] Review product features and data handling practices
- [ ] Meet with Product Manager on roadmap compliance needs
- [ ] Access compliance tracking and audit tools

### Week 2: Risk Assessment
- [ ] Review current features for compliance risks
- [ ] Meet with Developers on data collection and storage practices
- [ ] Collaborate with DevOps Engineer on security controls and audit logs
- [ ] Review terms of service, privacy policy, and user agreements
- [ ] Identify open-source licenses and IP considerations

### Week 3: Compliance Integration
- [ ] Define compliance approval gates for new features
- [ ] Provide security and privacy requirements to Developers
- [ ] Participate in design review with compliance perspective
- [ ] Review QA test plans for compliance validation
- [ ] Document compliance processes and training materials

---

## How to Use This Checklist
1. Select the section relevant to your role
2. Work through items in the suggested timeline (adjust as needed)
3. Check off items as you complete them
4. Ask questions and seek support when blocked
5. Review your progress with your manager at week 2 and week 4
